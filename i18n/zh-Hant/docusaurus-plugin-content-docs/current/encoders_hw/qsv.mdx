# QSV

:::caution 待審核
本條目內容尚待審核，其準確性可能需要進一步確認。 
:::

QSV（QuickSync Video）是 Intel 的硬體視訊編碼/解碼平台，整合於許多現代 CPU 的集成顯示處理器（iGPU）和 Arc 顯示卡中。應用程式可以將影片編碼、解碼及處理任務分配給 Intel 專用多媒體硬體上的專用媒體引擎，這通常相較於基於 CPU 的軟體編碼能提供顯著的效能提升。QSV 與類似的框架如 Nvidia 的 [NVENC](./nvenc.mdx) 和 AMD 的 [AMF](./amf.mdx) 競爭（自 Apple Silicon 轉型後，Intel Mac 上的 QSV 與 macOS 設備上的 Apple [VideoToolBox](./videotoolbox.mdx) 競爭）。

QSV 的主要目的是通過利用 Intel 顯示處理器中專用的固定功能硬體來加速視訊編碼、解碼和處理工作負載。這些專用硬體與通用計算單元不同，專為多媒體任務設計。QSV 旨在提供高效能的編碼/解碼性能，同時高效運作。

QSV 支援廣泛的視訊編碼格式，包括 [H.264](../video/AVC.mdx)、[H.265](../video/HEVC.mdx)、[VP9](../video/VP9.mdx)，以及最近在最新的獨立和集成 GPU 上支援的 [AV1](../video/AV1.mdx)。QSV 的一大優勢是其幾乎普遍存在於大多數現代 Intel CPU 的集成顯示處理器上，使得硬體加速的視訊編碼在各種系統上變得更加普及。應用程式可以通過 Intel Media SDK、VA-API 或與流行的多媒體框架如 [FFmpeg](../utilities/ffmpeg.mdx)、GStreamer 等的整合，輕鬆利用 QSV 加速。

使用 QSV 進行的硬體加速視訊編碼，通常在低壓縮效率下顯著提升編碼性能，相較於基於軟體的編碼解決方案，能夠實現更高的編碼速度，尤其是在 60-200 fps 的實時編碼場景中，對於需要即時編碼的應用（如實時串流、視訊會議和遊戲錄製）特別有利。

雖然 QSV 並不是為高度高效的離線檔案編碼設計的，當速度優於品質時，它的表現仍然值得注意。QSV 在壓縮效率方面幾乎總是優於 AMD 的 AMF，並且在競爭中與 Nvidia 的 NVENC 相當。
